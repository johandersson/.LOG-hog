# Suggested Future Security Enhancements for .LOG-hog

This document is not part of the committed codebase and should be ignored by version control. It collects ideas for future improvements that are not yet implemented.

## High Priority
- **Tamper Detection**: Alert on external file modifications.
- **Automated Security Audits**: Integrate static analysis (PMD, SpotBugs) into CI/CD.

## Medium Priority
- **Security Questions**: Optional 2FA-like recovery for password loss.
- **Hardware Security**: TPM integration for key storage.
- **Advanced Randomization**: Use time-based entropy sources for delays.

## Low Priority
- **Key Rotation**: Periodic key updates for encrypted files.
- **Multi-Factor Authentication**: Biometric support for unlocking.
- **Secure Deletion on Uninstall**: File wiping when uninstalling the app.

If you have additional suggestions, add them here for future review.
